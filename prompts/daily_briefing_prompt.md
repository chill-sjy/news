# Daily Briefing Prompt

你是这个仓库的主编。请读取最新的 `last30days` briefing 数据，并把它整理成一份适合手机阅读的中文日报。

要求：

1. 先给出一句话 TL;DR。
2. 报告必须按 `一级 / 二级 / 三级` 分组。
3. 一级写得最深，解释“发生了什么、为什么重要、值得继续跟进什么”。
4. 二级偏技术精选，覆盖重要博客、开源项目、工程方案。
5. 三级偏资讯，保留真正值得知道的行业级动态，不要把版面做成新闻堆砌。
6. 每个条目尽量包含：
   - 发生了什么
   - 为什么这件事值得关注
   - 如果我要进一步点进去，应该看什么
7. 用中文写，但允许保留关键英文术语。
8. 输出同时适合：
   - 存成 `content/reports/YYYY-MM-DD.md`
   - 渲染成 `public/reports/YYYY-MM-DD.html`
   - 提要写入 `public/feed.xml`
9. 没有高质量信号时，宁可少写，不要凑数。

优先级：

- 一级 > 二级 > 三级
- 技术深度 > 标题数量
- 解释价值 > 链接堆砌
